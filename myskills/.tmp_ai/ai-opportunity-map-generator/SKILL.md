# AI机会场景地图生成器 (AI Opportunity Map Generator)

此技能用于将用户提供的业务流程描述转化为高度结构化、且具备精美前端样式（HTML+CSS）的“AI机会场景地图”。用户不再需要处理抽象的 YAML 数据，而是可以直接在浏览器中查阅美观的可视化蓝图。

## 工作流

1. **理解业务流程**：审阅用户提供的业务或工作流描述。细致识别出核心阶段、用户关键动作、执行角色、接触点系统以及流程痛点。
2. **提取映射结构**：围绕两个核心问题对每一个动作环节进行解构：
   * 哪些任务属于“枯燥、重复性、低附加值”的数据搬运？
   * 哪些任务属于“复杂、耗费脑力、易出错”的决策分析？
3. **推演 AI 场景**：为这两个层面的痛点针对性地设计具备商业降本增效价值的落地方案。
4. **生成 HTML 地图**：根据参考文件中的样式规范和骨架，输出一个具备 Flexbox/CSS Grid 排版的、可滚动的单文件网页代码。

## 核心输出准则

1. **绝对前端就绪**：必须直接输出一个单页的 HTML（包含内部 `<style>` 样式表）。代码应该是完整合法的界面原型。
2. **拒绝空泛描述**：AI 机会场景不仅仅说“智能化系统”，必须使用“「目标用户」在「具体场景」下提供[XX]能力，以「XX收益」”的严格句式描述。
3. **输出纯净的代码**：输出可以包裹在 Markdown 的 `html` 代码块中，但代码内容应当直接可用，千万不要残留 YAML 遗迹，不写冗长的开场白和结语。
