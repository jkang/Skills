# Copyright Updater

此技能用于自动检测和更新项目中的版权信息，确保版权年份与当前年份保持一致。

## 使用场景

- 每年年初进行代码库维护时。
- 发现旧代码文件中包含过期的版权年份时。
- 用户明确要求检查项目版权标识时。

## 工作流程

1. **搜索版权标识**：使用 `grep` 或 `SearchCodebase` 搜索包含 `Copyright`、`©` 或 `(c)` 的字符串。
2. **提取年份**：识别标识后的年份（例如 `2023`, `2020-2024`）。
3. **验证年份**：检查年份是否为当前年份（2026）。
4. **自动更新**：
   - 如果是单一年份（如 `Copyright 2024`），更新为 `Copyright 2024-2026` 或直接更新为 `Copyright 2026`。
   - 如果是范围（如 `2020-2024`），更新为 `2020-2026`。
5. **提交更改**：完成更新后，自动执行 `git commit`。

## 指南

- 优先保留原始起始年份。
- 确保不破坏代码注释的格式。
- 在提交前进行验证，确保只修改了版权相关的文本。
